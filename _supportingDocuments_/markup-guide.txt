"\"はLaTeXのマークアップ指定。いろいろなケースがあるが、.texの中身と
PDFを見比べるとなんとなくわかる。"\abc is ..."のように書かれている場合、
「\abc は」のようにスペースを空けても空けなくても同じ結果になる(はず)なの
で、あまり気にしなくてよい。

\\						-> 改行の指定。とりあえず残す。

\chapter{xyz}, \section{xyz}, \subsection{xyz}
\paragraph{xyz}                                 -> 章、節、小節、段落の名前。訳す。


\textbf{xyz}                                    -> フォントの指定。物理的に太字を使うという指定。中身が訳せるものであれば訳す。
\emph{xyz}                                      -> 論理的な意味で強調表示するという指定。中身が訳せるものであれば訳す。後ほど日本語用マクロとして、太字にするように指定することも可能。
\st                                             -> Smalltalkになる。
\ind{xyz}                                       -> xyzが索引にのるので中身を訳す。
\emphsubind{xyz}, \subind{xyz}                  -> フォント指定と索引指定をまとめたもの。xyzが索引に乗るので訳せるものであれば訳す。
\indexmain{xyz}	  				-> 実際に意味があるのかどうかは不明だが、おそらくは訳さないのではなかろうか。

\ct{xyz}                                        -> \ctはコードに使うフォントを指定しているなので、中身が訳せるものであれば訳すが、おそらくは訳す必要のないコード断片などが多い

\figlabel{abc}, \tablabel{abc}, \chalabel{abc}
および
\figref{abc}, \tabref{abc}, \charef{abc}        -> 図、表、章などの参照番号に置き換えられる。中の文字列(abcのところ)は変更しない。
                                                  原文に、"\figref{abc} shows..."のように書かれていたら、
                                                  日本語の方は「図
						  \figref{abc}では...」
						  のように訳すと、「図4.3では...」のように出力される

\caption{xyz ....  \figref{abc}}                -> \begin{figure}などの中に、キャプションとして埋まっているので文章のところは訳す。
                                                  お尻についていることが多い\...label{abc}はそのままおしりにつけておく

\important{xyz}                                 -> xyzを訳す。
\ie                                             -> 「すなわち」という意味だが、現状では英語のようにi.e.と表示される。
                                                 　後ほどマクロを追加して、「すなわち」と置き換えられるようにすることができるので、そのまま残すのでよい。
\begin{code} ... \end{code}                     -> コード断片。コメント部分は訳すのが良いと思われる。
\begin{table} ... \end{table}                   -> 表。中身はだいたい他のところと同じ。&や\\には意味があるので気をつけて残す。および\captionの中身を訳す。
\begin{itemize} ... \end{itemize}		-> 箇条書き。もし、\itemはそのまま残せば良い。\item{xyz}のようなものがあれば、xyzのところは訳す。
